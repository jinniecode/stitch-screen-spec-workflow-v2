# 03. 기능 정의서 | TripTune
## 1. 기능 우선순위 기준
| 우선순위 | 기준 | 적용 예시 |
|---|---|---|
| Must | MVP 핵심 흐름에 반드시 필요한 기능 | 조건 입력, 취향 진단, 후보 추천, 일정 생성,
저장, 읽기 전용 공유, 로그인 |
| Should | 핵심 경험을 보조하지만 없어도 1차 사용은 가능한 기능 | 같은 타임슬롯 장소
교체, 저장 후 만족도 피드백 |
| Could | 후속 고도화로 넘길 수 있는 기능 | 전체 일정 재최적화, 고도화된 개인화, 공유
링크 로그인 정책 고도화 |
| Excluded | 이번 PRD 또는 MVP 범위에서 제외할 기능 | 동행자 선호 조율, 날씨 대안,
예약, 결제, 커뮤니티, 가계부, 해외 전 지역, 에이전트 |
