# 02. 메뉴 설계서 | TripTune
## 1. 메뉴 설계 목적
| 항목 | 내용 |
|---|---|
| 목적 | 사용자가 최소 입력으로 AI 여행지 후보를 받고, 선택한 여행지의 일정을 생성한 뒤
저장/공유까지 완료하는 MVP 흐름을 설계한다. |
| Gemini 반영 방향 | 다인원 조율과 실시간 날씨 대안은 MVP에서 제외하고,
“혼자 빠르게
일정을 만든 뒤 읽기 전용 링크로 공유”하는 경험에 집중한다. |
| MVP 설계 범위 | 조건 입력, 취향 진단, 여행지 후보 추천, 일정 자동 생성, 일정 저장, 읽기
전용 공유, 단일 슬롯 장소 교체, 저장 후 피드백 |
| 핵심 전환 | 첫 추천 완료 → 첫 일정 생성 완료 → AI 생성 일정 저장 → 읽기 전용 공유 링크
생성 |
| 설계 원칙 | 검색보다 선택, 복잡한 협업보다 빠른 생성/저장, 독립 메뉴보다 일정 상세
중심 액션, MVP 범위 초과 기능은 후순위 처리 |
## 2. IA 설계 원칙
| 원칙 | 설명 | 반영 위치 |
|---|---|---|
| 핵심 플로우 우선 | AI 추천 → 일정 생성 → 저장/공유를 최상위 흐름으로 배치 | 홈, 여행
만들기, 내 여행 |
| 1Depth 단순화 | 메인 탭은 [홈 / 여행 만들기 / 내 여행] 3개로 제한 | 1Depth / 2Depth 메뉴
트리 |
| 입력 이탈 최소화 | 취향 진단은 5문항 이내로 구성하고 건너뛰기를 허용 | 취향 진단 |
| 비교 가능한 추천 | 여행지 후보 3개를 추천 이유와 예상 비용 기준으로 비교 | 여행지 후보
추천 |
| 일정 상세 중심 액션 | 저장, 공유, 장소 교체, 피드백은 독립 메뉴가 아니라 일정 상세/저장
완료의 하위 액션으로 제공 | 일정 상세, 저장 완료 |
| 공유는 읽기 전용 | MVP 공유 링크 수신자는 일정 상세(공유 뷰)를 조회만 가능 | 일정
상세(공유 뷰) |
| MVP 제외 명확화 | 동행자 선호 수집/조율, 날씨 대안, 예약/결제, 커뮤니티, 가계부, 해외
전 지역, 에이전트는 제외 | MVP 제외 메뉴 |
## 3. 핵심 사용자 플로우
| 플로우 ID | 사용자 플로우 | 주요 화면 | 완료 기준 | 관련 지표 | MVP 상태 |
|---|---|---|---|---|---|
| UF-01 | 신규 사용자가 여행 조건을 입력하고 취향 진단을 완료하거나 건너뛴 뒤 여행지
후보 3개를 확인 | 홈, 여행 조건 입력, 취향 진단, 여행지 후보 추천 | 후보 추천 결과 노출 |
첫 추천 완료율 | 포함 |
| UF-02 | 사용자가 후보 중 1개를 선택하고 시간대별 일정을 자동 생성 | 여행지 후보 추천,
일정 생성 로딩, 일정 상세 | 일정 상세 화면 노출 | 첫 일정 생성 완료율 | 포함 |
| UF-03 | 사용자가 자동 생성된 일정을 저장하고 필요 시 카카오 로그인을 완료 | 일정 상세,
로그인, 저장 완료 | 저장 완료 | AI 생성 일정 저장률, 4주 내 재방문율 | 포함 |
| UF-04 | 사용자가 저장 완료 후 읽기 전용 공유 링크를 생성 | 저장 완료, 공유 설정, 일정
상세(공유 뷰) | 공유 링크 생성 | 공유 링크 생성률 | 포함 |
| UF-05 | 사용자가 특정 장소를 같은 타임슬롯 안에서 교체하고 이동 시간을 갱신 | 일정
상세, 장소 교체, 일정 상세 | 해당 슬롯 교체 및 이동 시간 갱신 | 일정 편집 액션 | 포함 |
| UF-06 | 동행자가 공유 링크에서 선호를 입력하고 AI 조율안을 확인 | 일정 상세(공유 뷰),
동행자 선호 입력, 조율 결과 | 선호 수집 및 조율안 노출 | 추천 만족도 | Excluded |
| UF-07 | 여행 중 날씨 변수 발생 시 대체 코스를 추천받아 반영 | 일정 상세, 날씨 대안 |
대체 코스 적용 | 일정 편집 액션 | Excluded |

## 6. 각 화면의 목적
| 화면명 | 화면 목적 | 사용자 기대 |
|---|---|---|
| 홈 | TripTune의 핵심 가치와 여행 만들기 CTA 제공 | 바로 여행 계획을 시작한다 |
| 여행 조건 입력 | 날짜, 예산, 출발지/현재 위치, 동행자, 이동 방식, 여행 분위기 수집 | 긴

여부를 선택한다 |

선택한다 |

공유하거나 만족도를 남긴다 |

| 일정 상세(공유 뷰) | 공유받은 사용자가 일정을 읽기 전용으로 확인 | 별도 조율 없이

| 일정 상세(공유 뷰) | 가능 | 가능 | 링크 보유자는 비로그인 읽기 가능 |
| 장소 교체 | 가능 | 가능 | 생성 직후 편집은 가능, 저장된 일정 편집은 로그인 필요 |
| 로그인 | 가능 | 이미 로그인 시 미노출 | 카카오 로그인만 제공 |

대비 큼 | Excluded |
