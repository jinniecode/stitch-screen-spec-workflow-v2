# 서비스 정책 설계서


## 1. 문서 개요


* 문서 목적: 메뉴 설계서와 기능 정의서를 기반으로 TripTune MVP 운영에 필요한 서비스 정책 기준을 정리한다.
* 작성 기준: 입력 문서에 명시된 내용은 근거 수준을 높게 반영하고, 명시되지 않은 내용은 가정 또는 추가 확인 필요 항목으로 분리한다.
* 입력 문서: docs/menu-structure.md, docs/feature-spec.md
* 적용 범위: 조건 입력, 취향 진단, 여행지 후보 추천, 일정 자동 생성, 일정 저장, 읽기 전용 공유, 같은 타임슬롯 장소 교체, 저장 후 피드백, 카카오 로그인.
* 검토 필요 영역: 법무 검토 필요, 개인정보 검토 필요, 위치정보 검토 필요, AI 고지 검토 필요, 정산/법무 검토 필요, 운영/법무 검토 필요, 사용자 고지 기준, 정책상 예외 허용 여부.
* 법령 고려 원칙:
  * 개인정보 보호법: 목적에 필요한 최소 개인정보 처리, 개인정보 처리방침 공개, 만 14세 미만 아동 개인정보 처리 시 법정대리인 동의 확인을 기준으로 삼는다.
  * 위치정보의 보호 및 이용 등에 관한 법률: 개인위치정보 수집·이용은 별도 동의, 목적·보유기간 고지, 목적 달성 후 파기, 위치정보 접근권한 최소화를 기준으로 삼는다.
  * 인공지능 발전과 신뢰 기반 조성 등에 관한 기본법: 생성형 AI 기반 서비스임을 사전에 쉽게 알아볼 수 있게 고지하는 방향으로 반영한다.
  * 전자상거래 등에서의 소비자보호에 관한 법률: 예약·결제·환불·정산을 제공하지 않는 MVP에서는 거래조건 오인 방지와 예상 비용 참고값 고지를 우선한다.
  * 정보통신망/플랫폼 운영 일반 원칙: 공유 링크 오남용, 권리침해, 삭제 요청은 고객 문의 기반 수동 검토와 링크 비활성화 가능성을 둔다. 운영/법무 검토 필요.


## 2. 서비스 개요


* 서비스 목적: 사용자가 최소 입력으로 AI 여행지 후보를 받고, 선택한 여행지의 일정을 생성한 뒤 저장 및 읽기 전용 공유까지 완료하도록 돕는다.
* 주요 사용자: 빠르게 국내 여행 후보와 일정을 만들고 저장·공유하려는 비로그인 또는 로그인 사용자. 추가 확인 필요: 미성년자 이용 가능 연령과 보호자 동의 정책.
* 주요 메뉴: 홈, 여행 조건 입력, 취향 진단, 여행지 후보 추천, 일정 생성 로딩, 일정 상세, 저장 완료, 공유 설정, 일정 상세(공유 뷰), 장소 교체, 로그인
* 핵심 기능: 여행 만들기 시작, AI 여행 조건 입력, 취향 진단, 여행지 후보 추천, 일정 자동 생성, 시간대별 일정 확인, 일정 저장, 읽기 전용 공유 링크 생성, 공유 일정 읽기 전용 열람, 같은 타임슬롯 장소 교체, 저장 후 추천 만족도 피드백, 카카오 로그인/사용자 식별
* 정책 설계 전제:
  * 메인 흐름은 홈, 여행 만들기, 내 여행 중심이며, 저장/공유 시점에는 로그인이 필요하다.
  * 여행 조건 입력, 취향 진단, 여행지 후보 추천, 일정 자동 생성, 일정 저장, 읽기 전용 공유가 MVP 핵심 범위다.
  * 공유 링크 수신자는 비로그인 상태에서도 읽기 전용으로 일정 상세를 조회할 수 있고 수정은 불가하다.
  * 예상 비용에는 교통비/식비/입장료만 포함하며 숙박비 제외 라벨을 표시한다.
  * 취향 진단은 5문항 이내이며 건너뛰기를 허용한다.
  * 장소 교체는 같은 타임슬롯에 한정하고 전체 일정 재최적화는 MVP 범위에서 제외한다.
  * MVP 제외 기능: 동행자 선호 수집, 동행자 조율안 확인, 날씨 대안 제안, 예약/결제, 커뮤니티/가계부/해외 전 지역/에이전트, 제휴 클릭 추적


## 3. 정책 설계 원칙


* 사용자 보호: AI 생성 결과, 예상 비용, 영업시간, 이동 시간은 실제와 다를 수 있음을 명확히 안내하고, 생성형 AI 기반 서비스임을 화면에서 사전 고지한다. 법무/AI 고지 검토 필요.
* 운영 가능성: MVP에서는 관리자 도구나 수동 보정 프로세스를 전제로 하지 않고, 실패 시 원본 유지·재시도·조건 수정 안내를 우선한다.
* 일관성: 비로그인 사용자는 추천과 생성까지 가능하되, 저장·공유처럼 계정 귀속이 필요한 행위는 카카오 로그인을 요구한다.
* 예외 처리 가능성: 지원 지역 외 입력, AI 실패, 링크 만료, 삭제, 권한 없음, 대체 장소 없음 등은 사용자에게 다음 행동을 안내한다.
* 법무/개인정보 검토 필요 영역 분리: 개인정보 수집·보관·삭제, 개인위치정보 별도 동의·파기, 카카오 식별 정보, 공유 링크 접근, AI 책임 고지, 비용 고지는 확정 전까지 검토 필요로 표시한다.


## 4. 메뉴별 정책 설계


### 홈


* 메뉴명: 홈
* 관련 기능: F-001 여행 만들기 시작
* 사용자 행동: 여행 만들기 CTA 선택
* 필요한 정책: 비로그인 접근, CTA 이동, 로딩 실패 안내
* 기본 정책: 비로그인/로그인 모두 접근 허용
* 예외 정책: 콘텐츠 로딩 실패 시 재시도 안내
* 운영 처리 기준: 운영자 개입 없음
* 사용자 안내 문구: 여행 조건을 입력하면 AI가 여행지 후보를 추천합니다.
* 검토 필요 여부: 아니오


### 여행 조건 입력


* 메뉴명: 여행 조건 입력
* 관련 기능: F-002 AI 여행 조건 입력
* 사용자 행동: 날짜, 예산, 출발지/현재 위치, 동행자, 이동 방식, 여행 분위기 입력 후 다음 선택
* 필요한 정책: 필수값 검증, 위치 권한 선택, 지원 지역 제한, 개인정보 검토
* 기본 정책: 필수값 입력 후 취향 진단으로 이동
* 예외 정책: 위치 거부·지오코딩 실패·지원 지역 외 입력 시 안내
* 운영 처리 기준: 반복 실패 문의는 FAQ/고객 문의로 처리
* 사용자 안내 문구: 현재 위치 제공은 선택이며, 입력 정보는 추천 생성에 사용됩니다. 개인정보 검토 필요.
* 검토 필요 여부: 예: 개인정보 검토 필요


### 취향 진단


* 메뉴명: 취향 진단
* 관련 기능: F-003 취향 진단
* 사용자 행동: 취향 문항 응답 또는 건너뛰기 선택
* 필요한 정책: 건너뛰기 허용, 기본 추천 전환
* 기본 정책: 5문항 이내 응답 또는 건너뛰기 허용
* 예외 정책: 건너뛰면 기본 추천으로 진행
* 운영 처리 기준: 운영자 개입 없음
* 사용자 안내 문구: 건너뛰면 추천 정확도가 낮아질 수 있습니다.
* 검토 필요 여부: 아니오


### 여행지 후보 추천


* 메뉴명: 여행지 후보 추천
* 관련 기능: F-004 여행지 후보 추천
* 사용자 행동: 후보 비교 후 1개 선택
* 필요한 정책: AI 응답 검증, 비용 범위 고지, 지원 지역 통제
* 기본 정책: 후보 3개와 추천 이유 및 예상 비용 표시
* 예외 정책: AI 실패·환각·비용 산정 불가 시 재시도/조건 수정 안내
* 운영 처리 기준: 반복 오류는 고객 문의로 접수
* 사용자 안내 문구: 예상 비용은 교통비/식비/입장료 기준이며 숙박비는 제외됩니다. 법무 검토 필요.
* 검토 필요 여부: 예: 법무 검토 필요


### 일정 생성 로딩


* 메뉴명: 일정 생성 로딩
* 관련 기능: F-005 일정 자동 생성
* 사용자 행동: 후보 선택 후 일정 만들기 선택
* 필요한 정책: 타임아웃, 라우팅 실패, 재시도 기준
* 기본 정책: 30초 내 일정 생성 시 상세로 이동
* 예외 정책: 타임아웃 또는 실패 시 이전 화면 복귀와 재시도
* 운영 처리 기준: 수동 보정 없음
* 사용자 안내 문구: 생성이 지연되면 조건을 수정하거나 다시 시도할 수 있습니다.
* 검토 필요 여부: 예: 운영 검토 필요
